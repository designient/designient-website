# Designient content voice guide

Internal reference for all user-facing copy (site, emails, course data, blogs).

## Em dashes

**Never use `—` (em dash) in user-facing content.**

| Context | Use instead |
|---------|-------------|
| Page titles / metadata | ` \| ` (e.g. `UI UX Design Pro \| Beginner to Pro`) |
| Week / module labels | Colon (`Week 1: Design Foundations`) |
| Clause break in body | Comma, period, or spaced hyphen ` - ` |
| Ranges in copy | En dash with spaces or words (`Weeks 1 to 2`) |

Run `npm run content:lint` before merging copy changes.

## Human voice (anti-AI)

Write like a mentor explaining the program to a serious student.

**Do:**
- Short sentences. One idea each.
- Specific numbers: cohort size, weeks, hours, tool names.
- Plain verbs: build, ship, review, apply, run.
- Name real outcomes: portfolio piece, case study, deployed workflow.

**Avoid:**
- comprehensive, leverage, seamless, cutting-edge, game-changer, empower
- delve, unlock, robust, elevate, transformative
- "in today's [industry/landscape/world]"
- Long chains of adjectives
- Passive filler ("It is important to note that…")

## E-E-A-T (Google people-first content)

**Experience:** Cohort format, mentor-led critique, what students actually produce.

**Expertise:** Name tools, weeks, deliverables. Tie claims to program structure.

**Authoritativeness:** Consistent stats with methodology footnotes where cited (e.g. placement rate among students who completed placement support).

**Trustworthiness:** Clear pricing, refund terms, contact info, no exaggerated guarantees.

## Claims

When stating placement rate, packages, or seat counts:
- Pair with who is included in the statistic, or
- Use "among students who completed placement support" where accurate.

## Blog posts

- Named author with role (mentor / curriculum lead)
- `updatedDate` when materially revised
- India-relevant examples where natural
- No generic future-tense trend pieces without sources
